# スケジュールボード

家のホワイトボードをそのままスマホに。家族みんなで書きこめて、B5サイズでホワイトボード風に印刷できる、子どものスケジュール管理アプリです。

- スマホ・PCのブラウザで動作(ホーム画面に追加すればアプリのように使えます)
- 家族共通の**合言葉**でログイン。誰が入力しても全員の画面にすぐ反映(Firebase)
- 電波がない場所でも入力OK。つながったときに自動で同期
- 「起きる」「就寝」の枠、緑のマグネット風ラベル(算数・国語・英語など)、チェックボックス、時刻、「なし」表示
- **B5(182×257mm)** で印刷。チェック欄は空欄で出るので、紙は子どもが自分でチェックできます

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` / `style.css` / `app.js` | アプリ本体 |
| `firebase-config.js` | ★唯一、自分用に書き換えるファイル |
| `firestore.rules` | Firestoreに貼るセキュリティルール |
| `manifest.json` / `icon-*.png` / `apple-touch-icon.png` | ホーム画面追加用 |

## セットアップ(初回のみ・15分ほど)

### 1. Firebaseプロジェクトを作る
1. https://console.firebase.google.com を開き、Googleアカウントでログイン
2. 「プロジェクトを作成」→ 名前は自由(例: `schedule-board`)。Google アナリティクスは**オフ**でOK

### 2. ログイン機能を有効にする
1. 左メニュー「構築」→「Authentication」→「始める」
2. 「メール / パスワード」を選び、**有効にする** → 保存

### 3. データベース(Firestore)を作る
1. 左メニュー「構築」→「Firestore Database」→「データベースを作成」
2. ロケーションは `asia-northeast1`(東京)がおすすめ →「**本番環境モード**」で作成
3. 作成後、「ルール」タブを開き、中身をこのフォルダの `firestore.rules` の内容に**まるごと置き換えて**「公開」

### 4. アプリの設定値をコピーする
1. 左上の歯車 →「プロジェクトの設定」→ 下の「マイアプリ」で **ウェブ(`</>`)** を選ぶ
2. アプリ名は自由。「Firebase Hosting」のチェックは**不要**
3. 表示される `firebaseConfig = { ... }` の中身を、このフォルダの **`firebase-config.js`** の同じ場所に貼り付けて保存

### 5. GitHub Pagesで公開する
1. GitHubで新しいリポジトリを作成(**Public**)
2. このフォルダのファイル一式をリポジトリの**ルート直下**にアップロード(git push でも、Webの「Add file → Upload files」でもOK)
3. リポジトリの Settings → Pages → 「Branch: `main` / `(root)`」を選んで Save
4. 数分後、`https://ユーザー名.github.io/リポジトリ名/` で開けます

### 6. FirebaseにPagesのドメインを登録する
Firebaseコンソール → Authentication → 「設定」タブ → 「承認済みドメイン」に `ユーザー名.github.io` を追加

### 7. 合言葉を登録する(公開後すぐに)
1. 公開したページを開く → 「初回セットアップ」を開き、家族で決めた合言葉(**6文字以上**・推測されにくいもの)を登録
2. 家族にURLと合言葉を共有。以後は各自がその合言葉でログインするだけ
3. スマホでは、ブラウザのメニューから「**ホーム画面に追加**」しておくとアプリのように起動できます

## 使い方

- **毎日の基本項目**: 新しい日を開くと、最初から「朝食・昼食・夕食・ピアノ・バイオリン」「算数(計算/テキスト)」「国語(漢字/文章題)」「英語(テキスト/単語)」が並びます。その日だけの変更は画面上でそのまま編集を。基本セット自体を変えたいときは `app.js` 冒頭の `DAY_TEMPLATE`(コメント付き)を書き換えます
- **日付**: 上部の `◀ ▶` で前後の日へ。日付をタップするとカレンダーで選べます
- **ラベル**: ラベル欄に「算数」などを入れると緑のマグネット風になります。下段の「追加:」ボタンからワンタップでも追加できます
- **項目**: チェックボックス・やること・時刻(`19:00〜` のような自由入力)。`↑↓` で並べ替え、`✕` で削除
- **なし**: その教科が今日はお休みのとき、「なし」ボタンで `→ なし` 表示に
- **メモ**: ラベルの下の小さな欄(例: `漢字・文章`)
- **前の日の内容をコピー**: 直近14日でいちばん新しい日の項目を、チェックを外した状態でコピー
- **同時編集**: 家族が同時に書きこめます。まったく同じ瞬間に別の端末で編集した場合は、あとから保存した内容が残ります

## 印刷(B5)

1. 右上の「B5印刷」をタップ
2. 印刷ダイアログで **用紙サイズ: B5**、倍率: 100%、**「背景のグラフィック」をオン**(緑ラベルを印刷するため)
3. チェック欄は空欄で印刷されるので、紙に貼ってチェックできます

スマホからも印刷できますが、レイアウトの確認はPCのChromeが確実です。

## 困ったとき

- **「設定エラー: メール/パスワードを有効に…」** → 手順2を確認
- **「合言葉がちがうか、まだ登録されていません」** → 未登録なら「初回セットアップ」から登録
- **保存されない / 権限エラー** → 手順3のルールが公開されているか、手順4の貼り付けが正しいか確認
- **合言葉を後で変えたい** → `firebase-config.js` の `FAMILY_EMAIL` をご自身の実在メールアドレスにしておくと、Firebaseコンソール(Authentication → ユーザー → パスワードを再設定)から変更用メールを受け取れます
  - ※コンソールで**ユーザーを削除して作り直すのはNG**です(データの保存先が変わり、今までの予定が見えなくなります)

## しくみ・費用

- データは Firestore の `families/{家族ID}/days/{日付}` に1日1件。セキュリティルールにより、ログインした家族だけが読み書きできます
- `firebase-config.js` の内容は公開されても問題ありません(保護はルール側で行う設計です)
- この規模なら Firebase の無料枠(Sparkプラン)で十分です
